## 1. Environment Setup

### Prerequisites
- WSL Ubuntu 24.04 installed
- ROS2 Jazzy installed ([Official Guide](https://docs.ros.org/en/jazzy/Installation.html))
- VS Code with WSL extension

### Source ROS2 Jazzy

Every terminal session must have ROS2 sourced before using any `ros2` commands:

```bash
source /opt/ros/jazzy/setup.bash
```

To make this permanent (runs automatically on every new terminal):

```bash
echo "source /opt/ros/jazzy/setup.bash" >> ~/.bashrc
source ~/.bashrc
```

### Install colcon (if missing)

```bash
sudo apt install python3-colcon-common-extensions
```

## 2. Workspace Structure

A ROS2 workspace is your development environment where packages and nodes live.

### Create the workspace

```bash
mkdir -p ~/ros2_ws/src
cd ~/ros2_ws
colcon build
source install/setup.bash
```

### Folder structure explained

```
ros2_ws/
├── src/          ← you create this — all your packages go here
├── build/        ← auto-generated by colcon build (intermediate files)
├── install/      ← auto-generated by colcon build (compiled executables)
└── log/          ← auto-generated by colcon build (build logs)
```

> **Note:** You only manually create `src/`. The other three folders are generated automatically when you run `colcon build`.

### Verify workspace is recognized

```bash
echo $AMENT_PREFIX_PATH
```

Expected output after building a package:
```
/home/yourname/ros2_ws/install/my_robot_package:/opt/ros/jazzy
```

If only `/opt/ros/jazzy` shows, either the workspace hasn't been built yet or hasn't been sourced.

## 3. Creating the Python Node

### 3.1 Create the Python package

```bash
cd ~/ros2_ws/src
ros2 pkg create my_robot_package \
  --build-type ament_python \
  --dependencies rclpy std_msgs
```

**Flags explained:**
- `--build-type ament_python` — tells ROS2 this is a Python package
- `--dependencies rclpy` — core Python library for writing ROS2 nodes
- `--dependencies std_msgs` — standard message types (String, Int32, etc.)

### 3.2 Package structure created

```
my_robot_package/
├── package.xml           ← package metadata and dependencies
├── setup.py              ← entry points for running nodes
├── setup.cfg             ← build configuration
├── resource/
│   └── my_robot_package
└── my_robot_package/
    ├── __init__.py
    └── my_first_node.py  ← your node lives here
```

### 3.3 Create the node file

```bash
cd ~/ros2_ws/src/my_robot_package/my_robot_package
touch my_first_node.py
chmod +x my_first_node.py
```

### 3.4 Python publisher node code

```python
#!/usr/bin/env python3
import rclpy
from rclpy.node import Node
from std_msgs.msg import String

class MyFirstNode(Node):
    def __init__(self):
        super().__init__('python_publisher')
        self.publisher_ = self.create_publisher(String, 'hello_topic', 10)
        self.create_timer(1.0, self.timer_callback)

    def timer_callback(self):
        msg = String()
        msg.data = 'Hello from Python node!'
        self.publisher_.publish(msg)
        self.get_logger().info(f'Publishing: {msg.data}')

def main(args=None):
    rclpy.init(args=args)
    rclpy.spin(MyFirstNode())
    rclpy.shutdown()

if __name__ == '__main__':
    main()
```

**Code explained:**
- `create_publisher(String, 'hello_topic', 10)` — creates a publisher on topic `hello_topic` with queue size 10
- `create_timer(1.0, ...)` — fires `timer_callback` every 1 second
- `publisher_.publish(msg)` — sends the message to the topic

### 3.5 Register entry point in `setup.py`

```python
entry_points={
    'console_scripts': [
        'my_first_node = my_robot_package.my_first_node:main',
    ],
},
```

### 3.6 Confirm `setup.cfg`

```
[develop]
script_dir=$base/lib/my_robot_package
[install]
install_scripts=$base/lib/my_robot_package
```

### 3.7 Build and run

```bash
cd ~/ros2_ws
colcon build --packages-select my_robot_package
source install/setup.bash
ros2 run my_robot_package my_first_node
```

## 4. Creating the C++ Node

### 4.1 Create the C++ package

```bash
cd ~/ros2_ws/src
ros2 pkg create my_cpp_package \
  --build-type ament_cmake \
  --dependencies rclcpp
```

**Flags explained:**
- `--build-type ament_cmake` — tells ROS2 this is a C++ package using CMake

### 4.2 Package structure created

```
my_cpp_package/
├── CMakeLists.txt   ← build instructions for the C++ compiler
├── package.xml      ← package metadata and dependencies
├── include/         ← header files go here
└── src/             ← your C++ node files go here
```

### 4.3 Create the node file

```bash
cd ~/ros2_ws/src/my_cpp_package/src
touch my_first_node.cpp
```

### 4.4 C++ subscriber node code

```cpp
#include "rclcpp/rclcpp.hpp"
#include "std_msgs/msg/string.hpp"
#include <chrono>
#include <functional>
#include <memory>
using namespace std::chrono_literals;

class MyFirstNode : public rclcpp::Node {
public:
    MyFirstNode() : Node("cpp_subscriber") {
        subscription_ = this->create_subscription<std_msgs::msg::String>(
            "hello_topic", 10,
            std::bind(&MyFirstNode::topic_callback, this, std::placeholders::_1));
    }

private:
    void topic_callback(const std_msgs::msg::String::SharedPtr msg) {
        RCLCPP_INFO(this->get_logger(), "I heard: '%s'", msg->data.c_str());
    }
    rclcpp::Subscription<std_msgs::msg::String>::SharedPtr subscription_;
};

int main(int argc, char * argv[]) {
    rclcpp::init(argc, argv);
    rclcpp::spin(std::make_shared<MyFirstNode>());
    rclcpp::shutdown();
    return 0;
}
```

**Code explained:**
- `create_subscription<std_msgs::msg::String>('hello_topic', ...)` — subscribes to `hello_topic`
- `topic_callback` — runs every time a message is received on the topic
- `std::placeholders::_1` — passes the received message to the callback

### 4.5 Update `CMakeLists.txt`

Add these lines **before** `ament_package()`:

```cmake
find_package(std_msgs REQUIRED)

add_executable(my_first_node src/my_first_node.cpp)
ament_target_dependencies(my_first_node rclcpp std_msgs)
install(TARGETS my_first_node
  DESTINATION lib/${PROJECT_NAME}
)
```

### 4.6 Update `package.xml`

Add `std_msgs` dependency:

```xml
<depend>rclcpp</depend>
<depend>std_msgs</depend>
```

### 4.7 Build and run

```bash
cd ~/ros2_ws
colcon build --packages-select my_cpp_package
source install/setup.bash
ros2 run my_cpp_package my_first_node
```

## 5. Publisher & Subscriber Communication

The Python node publishes messages and the C++ node subscribes to them through a shared **topic** called `hello_topic`.

```
Python Node (publisher)
        |
        |  publishes String msg
        ▼
   [ hello_topic ]     ← ROS2 topic (message channel)
        |
        |  receives String msg
        ▼
C++ Node (subscriber)
```

- The publisher does NOT need to know the subscriber exists
- The subscriber does NOT need to know the publisher exists
- ROS2 connects them automatically through the topic name

## 6. Running the Nodes

Open **two terminals** and run one node in each:

**Terminal 1 — C++ Subscriber:**
```bash
source ~/ros2_ws/install/setup.bash
ros2 run my_cpp_package my_first_node
```

**Terminal 2 — Python Publisher:**
```bash
source ~/ros2_ws/install/setup.bash
ros2 run my_robot_package my_first_node
```

**Expected output in Terminal 1 (subscriber):**
```
[INFO] [cpp_subscriber]: I heard: 'Hello from Python node!'
[INFO] [cpp_subscriber]: I heard: 'Hello from Python node!'
```

**Expected output in Terminal 2 (publisher):**
```
[INFO] [python_publisher]: Publishing: Hello from Python node!
[INFO] [python_publisher]: Publishing: Hello from Python node!
```

Stop either node with **Ctrl + C**.

## 7. Errors Encountered & Fixes

### Error 1: `ros2: command not found`
**Cause:** ROS2 Jazzy not sourced.
**Fix:**
```bash
source /opt/ros/jazzy/setup.bash
```

### Error 2: `colcon build` shows `0 packages finished`
**Cause:** No packages exist in `src/` yet.
**Fix:** Create a package first, then build.

### Error 3: Package not found after build
**Cause:** Workspace not sourced after building.
**Fix:**
```bash
source install/setup.bash
```

### Error 4: `ament_target_dependencies() std_msgs was not found`
**Cause:** `find_package(std_msgs REQUIRED)` missing in `CMakeLists.txt`.
**Fix:** Add `find_package(std_msgs REQUIRED)` before `add_executable()`.

### Error 5: `Duplicate package names not supported`
**Cause:** Two copies of the same package exist in the workspace (e.g. one in `src/` and one inside a cloned repo).
**Fix:** Delete the duplicate:
```bash
rm -rf ~/ros2_ws/src/internship2026/emmanuelmumo/my_robot_package
```

### Error 6: Package pushed as Git submodule
**Cause:** The package folder had its own `.git` folder inside it.
**Fix:** Remove the nested `.git` before copying:
```bash
rm -rf my_robot_package/.git
```

### Error 7: `The generic dependency on 'rclcpp' is redundant`
**Cause:** `rclcpp` declared twice in `package.xml`.
**Fix:** Remove the duplicate `<depend>rclcpp</depend>` line.

## 8. Branch Structure

Each node is pushed on its own branch under your personal folder:

```
internship2026/
├── main                        ← instructor's base (do not touch)
├── python_node                 ← your Python publisher
│   └── emmanuelmumo/
│       └── my_robot_package/
└── cpp_node                    ← your C++ subscriber
    └── emmanuelmumo/
        └── my_cpp_package/
```

### How to push each branch

**Python node (`python_node` branch):**
```bash
cd ~/ros2_ws/src/internship2026
git checkout main && git pull origin main
git checkout -b python_node
mkdir emmanuelmumo
cp -r ~/ros2_ws/src/my_robot_package emmanuelmumo/
rm -rf emmanuelmumo/my_robot_package/.git
git add emmanuelmumo/
git commit -m "Day3: Add first ROS2 Python node"
git push -u origin python_node
```

**C++ node (`cpp_node` branch):**
```bash
cd ~/ros2_ws/src/internship2026
git checkout main && git pull origin main
git checkout -b cpp_node
cp -r ~/ros2_ws/src/my_cpp_package emmanuelmumo/
rm -rf emmanuelmumo/my_cpp_package/.git
git add emmanuelmumo/
git commit -m "Day3: Add first ROS2 C++ node"
git push -u origin cpp_node
```

> **Key rule:** Always branch from `main` and always remove `.git` from copied packages to avoid submodule issues.
